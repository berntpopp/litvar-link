# Phase 2 — CI/CD Implementation Plan

> Historical record — this document records the plan as of its date. Current behavior is defined
> by implemented code, standards, release evidence, and tests.

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the sibling house-style GitHub Actions CI/CD stack (CI, Docker, security, container-security, release) plus Dependabot to `litvar-link`, with every action SHA-pinned and CI delegating to the P0 Makefile.

**Architecture:** CI does no logic itself — it delegates to `make ci-local` + `make test-cov` (and `make docker-*-config` for compose validation), so the workflow stays a thin, vetted wrapper. Every action is SHA-pinned with a `# vX` version comment (copied verbatim from the `autopvs1-link` reference), each workflow declares top-level `permissions: contents: read` and a `concurrency` group with `cancel-in-progress: true`, and the `quality` job runs a `python-version: ["3.12", "3.13"]` matrix (test-what-you-ship).

**Tech Stack:** GitHub Actions, uv, Trivy, CodeQL, Dependabot.

**Prerequisite:** P0 (Makefile targets `ci-local`/`test-cov`/`docker-prod-config`/`docker-npm-config` and a committed `uv.lock` must exist). The four `docker/docker-compose*.yml` overlays already exist in litvar's `docker/`; the `docker/Dockerfile` is modernized to a uv-based build in **Task 0** of this phase, before any image is built. The `.env.npm.example` template is still named that through P0–P2 (P3 renames it to `.env.docker.example`).

## Task 0 — Modernize `docker/Dockerfile` to uv-based multi-stage

**Files:** Modify `docker/Dockerfile`

**Why this runs first:** the current Dockerfile uses `pip install -e .[production]` on `python:3.11-slim`. After P0 the project is `requires-python>=3.12` with a committed `uv.lock` and a PEP 735 dev group, so the old Dockerfile would fail to install on a 3.11 base. Tasks 2/4/5 build this image, so it must be modernized before they can pass. Base is `python:3.13-slim` (inside the CI 3.12/3.13 matrix → we test the runtime we ship).

- [ ] Replace the entire contents of `docker/Dockerfile` with EXACTLY the following (uv-based, ported from the vetted `autopvs1-link/docker/Dockerfile`, retargeted to `litvar_link`, base `python:3.13-slim`, healthcheck on litvar's `/api/health/`):

```dockerfile
# Multi-stage Dockerfile for LitVar-Link.

# --- Builder ---
FROM python:3.13-slim AS builder

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PIP_NO_CACHE_DIR=1 \
    PIP_DISABLE_PIP_VERSION_CHECK=1 \
    UV_COMPILE_BYTECODE=1 \
    UV_LINK_MODE=copy

RUN apt-get update && apt-get install -y --no-install-recommends \
    build-essential \
    curl \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/*

RUN python -m venv /opt/venv
ENV VIRTUAL_ENV="/opt/venv" \
    PATH="/opt/venv/bin:$PATH"

WORKDIR /app
COPY uv.lock pyproject.toml README.md ./

RUN pip install --upgrade pip uv && \
    uv sync --frozen --no-dev --active

# --- Production ---
FROM python:3.13-slim AS production

ENV PYTHONDONTWRITEBYTECODE=1 \
    PYTHONUNBUFFERED=1 \
    PATH="/opt/venv/bin:$PATH" \
    PYTHONPATH="/home/app/web" \
    LITVAR_LINK_HOST=0.0.0.0 \
    LITVAR_LINK_PORT=8000

RUN apt-get update && apt-get install -y --no-install-recommends \
    curl \
    ca-certificates \
    && rm -rf /var/lib/apt/lists/* \
    && apt-get clean

RUN groupadd --system app && \
    useradd --system --gid app --home /home/app --create-home app

COPY --from=builder /opt/venv /opt/venv

WORKDIR /home/app/web

COPY --chown=app:app ./litvar_link ./litvar_link
COPY --chown=app:app ./server.py ./mcp_server.py ./pyproject.toml ./README.md ./
COPY --chown=app:app ./docker/gunicorn_conf.py ./

RUN /opt/venv/bin/pip install -e . --no-deps

USER app

EXPOSE 8000

HEALTHCHECK --interval=30s --timeout=10s --start-period=30s --retries=3 \
    CMD curl -fsS http://localhost:8000/api/health/ || exit 1

CMD ["gunicorn", "-c", "gunicorn_conf.py", "litvar_link.app:app"]
```

```yaml
    branches:
      - main

  cancel-in-progress: true

    name: Format, lint, typecheck, tests, and coverage
    runs-on: ubuntu-latest
    strategy:

        with:
          enable-cache: true
          version: "0.8.7"

      - name: Install dependencies
        run: uv sync --group dev --frozen

      - name: Run local CI checks
        run: make ci-local

      - name: Run coverage
        run: make test-cov
```

**Files:** `.github/workflows/security.yml`

```yaml
name: Security

on:
  pull_request:
  push:
    branches:
      - main
  schedule:
    - cron: "17 3 * * 1"

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read

jobs:
  codeql:
    name: CodeQL
    runs-on: ubuntu-latest
    if: ${{ !github.event.repository.private }}
    permissions:
      actions: read
      contents: read
      security-events: write

    steps:
      - name: Checkout
        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6

      - name: Initialize CodeQL
        uses: github/codeql-action/init@ed410739ba306e4ebe5e123421a6bd694e494a2b # v4

          languages: python
          build-mode: none

      - name: Analyze
        uses: github/codeql-action/analyze@ed410739ba306e4ebe5e123421a6bd694e494a2b # v4

  dependency-review:
    name: Dependency review
    runs-on: ubuntu-latest
    if: github.event_name == 'pull_request'
    permissions:
      contents: read
      pull-requests: read

    steps:
      - name: Checkout
        uses: actions/checkout@de0fac2e4500dabe0009e67214ff5f5447ce83dd # v6

      - name: Dependency Review
        uses: actions/dependency-review-action@a1d282b36b6f3519aa1f3fc636f609c47dddb294 # v5.0.0
```
